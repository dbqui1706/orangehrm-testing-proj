# **End-to-End Automation Project (bắt đầu làm bài tập nhóm)**


### **Mục tiêu học tập**

- Áp dụng toàn bộ kiến thức về automation testing đã học.
- Thực hành kiểm thử **một quy trình hoàn chỉnh** (end-to-end).
- Biết cách **lập Test Plan**, tổ chức test suite, chạy automation, tạo báo cáo tổng hợp.
- Phát triển kỹ năng trình bày & bảo vệ sản phẩm test.

---

### **Yêu cầu công việc**

1. **Chọn Website / Ứng dụng Demo**
    - Có nhiều chức năng liên quan nhau (login → thực hiện 1 hành động → logout).
    - [OrangeHRM Demo](https://opensource-demo.orangehrmlive.com/) (login → search employee → logout).
2. **Thiết kế Test Plan** (2–3 trang)
    - Mục tiêu kiểm thử.
    - Phạm vi kiểm thử.
    - Chức năng cần test.
    - Danh sách test case chính.
    - Tiêu chí pass/fail.
3. **Viết Test Suite (5–7 test case)**
    - Bao gồm:
        - **Positive case** (luồng bình thường).
        - **Negative case** (sai dữ liệu, edge case).
        - **Integration/flow case** (End-to-End).
    - Mỗi test case: ID, mô tả, input, expected output.
4. **Thực hiện Automation Test**
    - Dùng công cụ Selenium WebDriver (Python).
    - Tạo Test Suite, chạy toàn bộ test case.
    - Ghi lại kết quả (Pass/Fail).
5. **Báo cáo Test**
    - Tóm tắt Test Plan.
    - Bảng kết quả automation test (có screenshot).
    - Phân tích lỗi (nếu có).
    - Kết luận: automation testing đã tiết kiệm gì so với manual testing.
